# DDD Diagnostic Report Expectations

This directory documents expected report semantics for the Sales/Billing
Customer DDD diagnostic fixture.

The fixture path is:

```text
examples/casegraphen/ddd/domain-model-design/sales-billing-customer.case.space.json
```

Expected native CaseGraphen report signals after importing with revision
`revision:ddd-sales-billing-imported`:

| Command | Expected signal |
| --- | --- |
| `space validate` | `result.validation.valid == true` |
| `space reason` | `result.evaluation.status == "blocked"` |
| `obstruction list` | contradiction, missing evidence, and review-required records are present |
| `completion candidates` | anti-corruption mapping, missing-evidence, missing-review, and contradiction-resolution candidates are present |
| `invariant check` | workshop notes are accepted evidence; AI diagnostics remain unreviewed inference |
| `projection apply` | requested projection information loss is visible |
| `invariant close-check` | `result.close_check.closeable == false` |

Store paths, checksums, and retargeted revision IDs are generated by the local
run, so reference JSON files are not checked in here.
