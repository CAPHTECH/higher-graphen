# Architecture Reference Workflow

This directory contains the compact Architecture Product end-to-end reference.
The scenario keeps MCP out of scope and uses only the checked-in CLI/runtime
surfaces.

## Files

- `architecture-reference.input.json`: bounded v1 architecture input with
  services, an API, a database, a requirement, a test, and one unreviewed
  inferred API candidate.
- `casegraphen-reference.case.graph.json`: file-based CaseGraphen graph that
  records the reference workflow cases and review record.
- `casegraphen-reference.coverage.policy.json`: coverage policy for the
  reference graph.
- `casegraphen-reference.projection.json`: AI/audit projection request for the
  reference graph.
- `reports/*.json`: deterministic report outputs generated by the commands
  below.

## Commands

Run from the repository root:

```sh
mkdir -p examples/architecture/reference/reports

cargo run -q -p highergraphen-cli -- architecture input lift \
  --input examples/architecture/reference/architecture-reference.input.json \
  --format json \
  --output examples/architecture/reference/reports/architecture-input-lift.report.json

cargo run -q -p highergraphen-cli -- architecture smoke direct-db-access \
  --format json \
  --output examples/architecture/reference/reports/architecture-direct-db-access-smoke.report.json

cargo run -q -p highergraphen-cli -- completion review accept \
  --input examples/architecture/reference/reports/architecture-direct-db-access-smoke.report.json \
  --candidate candidate:billing-status-api \
  --reviewer reviewer:architecture-lead \
  --reason "Billing owns the API boundary for status reads." \
  --reviewed-at 2026-04-25T00:00:00Z \
  --format json \
  --output examples/architecture/reference/reports/completion-review-accepted.report.json

cargo run -q -p casegraphen -- validate \
  --input examples/architecture/reference/casegraphen-reference.case.graph.json \
  --format json \
  --output examples/architecture/reference/reports/casegraphen-validate.report.json

cargo run -q -p casegraphen -- coverage \
  --input examples/architecture/reference/casegraphen-reference.case.graph.json \
  --coverage examples/architecture/reference/casegraphen-reference.coverage.policy.json \
  --format json \
  --output examples/architecture/reference/reports/casegraphen-coverage.report.json

cargo run -q -p casegraphen -- project \
  --input examples/architecture/reference/casegraphen-reference.case.graph.json \
  --projection examples/architecture/reference/casegraphen-reference.projection.json \
  --format json \
  --output examples/architecture/reference/reports/casegraphen-project.report.json
```

## Review Rules

- Accepted input facts are lifted as cells and incidences with accepted
  provenance.
- Inferred structures remain completion candidates until an explicit
  completion review command accepts or rejects one candidate.
- The source candidate snapshot remains unreviewed in the review report; the
  accepted or rejected outcome is recorded separately.
- Human, AI-agent, and audit projections must declare information loss and
  preserve source IDs.
