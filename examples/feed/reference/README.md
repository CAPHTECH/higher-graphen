# Feed Reference Workflow

This directory contains a compact HigherGraphen Feed reference workflow. The
scenario is a simplified RSS Reader example that uses a checked-in JSON fixture
instead of network fetching, scheduling, persistent read state, raw RSS/Atom
parsing, or a UI.

## Files

- `feed-reader.input.json`: bounded v1 feed input with official, community,
  blog, and aggregator source contexts.
- `reports/feed-reader.report.json`: deterministic report generated by the
  `highergraphen feed reader run` command below.

## Command

Run from the repository root:

```sh
mkdir -p examples/feed/reference/reports

cargo run -q -p highergraphen-cli -- feed reader run \
  --input examples/feed/reference/feed-reader.input.json \
  --format json \
  --output examples/feed/reference/reports/feed-reader.report.json
```

## Review Rules

- Source feeds and entries are lifted as accepted source-indexed observations.
- Topic and event cells inferred from hints remain unreviewed.
- Completion candidates remain unreviewed until an explicit review workflow is
  added.
- Obstructions describe feed-content conflicts and do not make the CLI command
  fail.
- Timeline, topic digest, and audit projections declare information loss and
  preserve source IDs.
