# Feed Product

The Feed Product is a compact reference product for using HigherGraphen as a
feed and RSS reader analysis workflow.

## Purpose

HigherGraphen Feed is a source-indexed observation space with temporal
evolution, correspondence, completion, obstruction, and projection. It lets a
runtime group related entries, flag missing viewpoints, detect conflicts
between sources, and project the result into timeline, topic, and audit views.

This first contract is intentionally bounded. It uses checked-in JSON fixtures
instead of raw RSS or Atom XML. Network fetching, scheduling, persistent read
state, UI, marketplace packaging, and production-grade feed parsing are outside
the initial example.

## Interpretation

HigherGraphen Feed maps feed reader vocabulary onto HigherGraphen primitives.

| HigherGraphen primitive | Feed interpretation |
| --- | --- |
| Source-indexed Observation Space | The full bounded feed fixture, indexed by source feed and entry time. |
| Context / Source Space | A feed source such as an official status feed, engineering blog, community forum, or aggregator. |
| Observation Cell | A feed entry captured from one source at a published time. |
| Temporal Evolution | Published and updated timestamps ordering entries in a timeline. |
| Topic / Event Cell | Runtime-inferred grouping cells for related entries, incidents, releases, or announcements. |
| Correspondence | A same-topic, duplicate, follow-up, or counterpoint relationship between entries. |
| Completion Candidate | A missing official source, missing counterpoint, missing metadata, or missing timeline context. |
| Obstruction | A feed/content conflict, timestamp conflict, broken source reference, or topic mismatch. |
| Projection | Timeline, topic digest, and audit views derived from the observed and inferred structure. |

## Input Contract

`schemas/inputs/feed-lift.input.schema.json` defines
`highergraphen.feed.input.v1`. The matching fixture is
`schemas/inputs/feed-lift.input.example.json`.

The input contract contains:

- `source`: provenance for the bounded JSON fixture.
- `space`: the observation space for the feed analysis run.
- `source_feeds`: source-indexed contexts such as official, community, and
  aggregator feeds.
- `entries`: observed feed entries. Each entry is an observation cell attached
  to one source feed and one or more topic identifiers.
- `correspondence_hints`: deterministic grouping hints for same-topic,
  duplicate, follow-up, or counterpoint relationships.
- `completion_hints`: deterministic missing-structure hints for official
  sources, counterpoints, metadata, timeline context, or source feeds.
- `obstruction_hints`: deterministic conflict hints that the runtime should
  preserve as obstructions.
- `projection_requests`: requested timeline, topic digest, and audit
  projections.

The fixture deliberately includes:

- A same-topic relationship between an official status entry and a community
  report about API latency.
- A duplicate relationship between an engineering blog entry and an aggregator
  entry about a webhook signing change.
- A missing official-source completion hint for the community latency report.
- A missing counterpoint completion hint for the official webhook signing
  announcement.
- A feed/content conflict obstruction hint where official status says an
  incident is resolved while community reports continued errors.

## Report Contract

`schemas/reports/feed-reader.report.schema.json` defines the runtime report
envelope, `highergraphen.feed.reader.report.v1`. The deterministic example is
`schemas/reports/feed-reader.report.example.json`, generated by:

```sh
highergraphen feed reader run \
  --input schemas/inputs/feed-lift.input.example.json \
  --format json
```

The report contract includes:

- The source fixture and lifted source feeds.
- Observed entry cells and inferred topic/event cells.
- Runtime-emitted correspondences, completion candidates, and obstructions.
- Timeline, topic digest, and audit projections with explicit information
  loss.

## Reference Workflow

The runnable reference lives in
[`examples/feed/reference`](../../examples/feed/reference/README.md). It copies
the bounded fixture into a product example path and generates
`reports/feed-reader.report.json` with:

```sh
cargo run -q -p highergraphen-cli -- feed reader run \
  --input examples/feed/reference/feed-reader.input.json \
  --format json \
  --output examples/feed/reference/reports/feed-reader.report.json
```

## MVP Validation Goal

This scenario is successful when HigherGraphen can:

1. Lift the bounded JSON fixture into a source-indexed observation space.
2. Preserve entries as observation cells with source and time provenance.
3. Infer topic or event grouping cells from correspondence hints.
4. Emit missing official-source and missing counterpoint completion candidates.
5. Emit the feed/content conflict as an obstruction.
6. Project the result into timeline, topic digest, and audit views.
